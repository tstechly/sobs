Data Model & OTEL Compatibility

SOBS stores all telemetry in ClickHouse-compatible tables that align with the OpenTelemetry (OTEL) semantic conventions and are designed to support future migration to Clickstack without data loss.

  • otel_logs — Logs and log-derived events (errors, RUM exceptions)
  • otel_traces — Distributed trace spans (including AI transparency spans)
  • otel_metrics_gauge / sum / histogram — OTEL metric data points
  • hyperdx_sessions — Real User Monitoring (RUM) session events
  • sobs_record_tags — Record-level tags (manual and auto-tagged), keyed by stable record ID
  • sobs_tag_rules — Auto-tagging rules evaluated at ingest time
  • sobs_anomaly_rules — Anomaly detection threshold/composite rules
  • sobs_ai_settings — AI/LLM endpoint and safety configuration
  • sobs_agent_rules — Automated agent flow trigger rules
  • sobs_agent_runs — History of agent flow executions and results

Tags are stored out-of-band in sobs_record_tags, keeping the core OTEL tables schema-compatible with upstream tooling. The stable RecordId is an MD5 of (ServiceName, Timestamp, TraceId, SpanId) for log-type records, and (TraceId, SpanId) for trace spans (including AI spans) — ensuring deterministic join keys across migrations.
