Auto-Created Issues

Work items are GitHub issues created automatically when an agent rule fires. Each item links to the original GitHub issue.

Filter & Browse

Filter by service, agent rule, action type (issue or issue + Copilot), status (open/closed), and date range.

Copilot Tasks

When action type is Issue + Copilot, the agent also assigns the issue to GitHub Copilot for automated resolution.

Filter Controls
Control Description
Service Filter to work items created for a specific service.
Agent Rule Filter to work items created by a specific agent rule.
Action Type Issue only — creates a GitHub issue. Issue + Copilot — creates an issue and assigns it to Copilot for automated fix.
Status Open or Closed GitHub issue state.
From / To Date range for when the work item was created.
Agent Rules & Work Item Creation

Work items are created when a configured agent rule detects a matching condition (for example, a spike in error rate or a new error pattern).

  • Configure agent rules on the Agent Settings page.
  • Each rule specifies the trigger condition, target GitHub repository, and action type.
  • A cooldown period prevents duplicate issues from being created for the same event.
GitHub API credentials must be configured in AI Settings for issue creation to work.
