Kubernetes Health Setup

Deploy in-cluster OpenTelemetry collectors to observe your Kubernetes cluster health via standard OTEL signals.

Back to Health View
What You Need
  • Kubernetes cluster (1.20+)
  • kubectl access to the cluster
  • Helm 3.x installed
  • SOBS running with OTLP HTTP receiver enabled (port 4318)
What You Get
  • Node status & readiness (from kubeletstats)
  • Pod phases & restart counts (from hostmetrics)
  • Deployment desired/ready replicas (from k8s_cluster)
  • Namespace-level resource aggregates
Installation Steps
  1. Add the OpenTelemetry Helm repository:
    helm repo add open-telemetry https://open-telemetry.github.io/opentelemetry-helm-charts
    helm repo update
  2. Install cluster-wide collector (Deployment):
    Handles Kubernetes API metadata, events, and cluster-level metrics.
    helm install sobs-k8s-deployment \
      open-telemetry/opentelemetry-collector \
      -f k8s/otel-k8s-deployment.yaml
  3. Install node-level collector (DaemonSet):
    Runs on every node to collect kubelet metrics, host metrics, and container logs.
    helm install sobs-k8s-daemonset \
      open-telemetry/opentelemetry-collector \
      -f k8s/otel-k8s-daemonset.yaml
  4. Verify collectors are running:
    kubectl get pods -o wide | grep -i otel
  5. Enable in SOBS:
    • Navigate to Settings → Kubernetes
    • Check the "Enable Kubernetes Health View" checkbox
    • Save settings
    • Data will appear in Kubernetes Health dashboard as metrics flow in
Collector Architecture
Cluster-Wide Deployment (Single Pod)

Runs once in the cluster (in the OpenTelemetry Collector namespace).

  • kubernetesAttributes: Enriches all signals with pod/node/cluster context
  • kubernetesEvents: Collects Kubernetes API events (pod created, deployment updated, etc.)
  • clusterMetrics: Gathers cluster-level state (nodes, namespaces, deployments)
Node-Level DaemonSet (One Pod Per Node)

Runs on every node in the cluster (via DaemonSet).

  • kubeletMetrics: Reads pod resource usage (CPU, memory) directly from kubelet
  • hostMetrics: Collects host-level metrics (disk, network, processes)
  • Container Logs: Gathers logs from running containers via CRI runtime
Data Flow:
Kubernetes API ← Collectors → OTLP HTTP (port 4318) → SOBS ClickHouse
Dashboard Data Mapping
Dashboard Section Data Source OTEL Signal
Nodes kubeletstats on each node k8s.node.condition_ready
Pods kubeletMetrics + container logs k8s.pod.name, k8s.container.restart_count
Deployments clusterMetrics + kubernetesEvents k8s.deployment.name, k8s.container.ready
Namespaces clusterMetrics k8s.namespace.name
Troubleshooting
Collectors not starting?

Check logs and resource requests:

kubectl logs -l app.kubernetes.io/name=opentelemetry-collector
No data appearing in SOBS?

Verify OTLP endpoint is reachable from cluster:

kubectl run -it --rm debug --image=curlimages/curl --restart=Never -- curl -v http://sobs:4318/v1/metrics
Wrong SOBS endpoint?

Update the Helm values endpoint field and upgrade:

helm upgrade sobs-k8s-deployment ... --set-string endpoint=http://your-sobs-ip:4318
Settings page doesn't show it enabled?

Check that SOBS is receiving metrics and has processed them. Initial data flow takes 30-60s.

mode: deployment
image:
  repository: otel/opentelemetry-collector-contrib
  tag: 0.123.0

presets:
  kubernetesAttributes:
    enabled: true
    extractAllPodLabels: true
    extractAllPodAnnotations: true
  kubernetesEvents:
    enabled: true
  clusterMetrics:
    enabled: true

exporters:
  otlphttp/sobs:
    endpoint: http://sobs:4318
    compression: gzip

service:
  pipelines:
    logs:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlphttp/sobs]
    metrics:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlphttp/sobs]

mode: daemonset
image:
  repository: otel/opentelemetry-collector-contrib

    enabled: true
