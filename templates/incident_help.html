Incident View Help

Correlated cross-signal incident timeline: primary event, related errors, logs, spans, and anomaly state.

Back to Errors
Open an Incident

Click the Incident View button on an error card or trace detail to open the incident timeline for that event.

Primary Event

The top card shows the anchor event (error, trace span, or RUM session) that the incident window is centered on.

Related Signals

Correlated errors, log lines, trace spans, metric anomalies, and work item links are aggregated in the same time window below the primary event.

Time Window

The incident view aggregates related signals within a configurable window around the primary event timestamp.

  • Default window: ±30 minutes around the primary event.
  • Maximum window: ±180 minutes.
  • Pass window_minutes as a query parameter to adjust the window size.
  • The from_ts and to_ts parameters override the automatic window calculation.
Correlated Signals
Section Description
Primary Event The anchor error, trace span, or RUM session that triggered the incident view.
Related Errors Other errors from the same service within the incident window, up to a configured limit.
Related Logs Log lines from the same service and time window. Useful for tracing the error path.
Related Spans Trace spans from the same trace ID or service within the window.
Anomaly State Active metric anomalies for the service at the time of the incident.
Work Items GitHub issues linked to the same trace ID or error fingerprint.
How to Navigate Here
  • From Errors — expand an error accordion and click Incident View.
  • From Traces — open a trace detail and click Incident View in the span header.
  • Direct URL — /incident?trace_id=<id> or /incident?error_id=<id>.
