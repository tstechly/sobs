Anomaly Chart

The main chart shows the signal value over the selected window with threshold bands overlaid. Anomalous regions are highlighted in orange or red.

Related Telemetry

Use the Open Related Logs / Traces / Errors button to jump to the corresponding telemetry page pre-filtered to the same service and time window.

Seasonal Baselines

Seasonal rules display per-bucket expected ranges on the chart. Hover a data point to see the bucket baseline for that time slot.

Anomaly States
State Meaning
OK Signal is within all rule thresholds for the current window.
Warning Signal has crossed at least one warning threshold but not a critical threshold.
Critical Signal has crossed at least one critical threshold. Investigate immediately.
No rules No metric rules are configured for this signal. State cannot be evaluated.
Seasonal Rules

Seasonal rules are auto-generated from historical signal data. They store per-bucket thresholds (e.g. one bucket per hour-of-week) and compare each new data point against its historical baseline.

  • The Seasonal chart overlay shows the expected range band for each data point.
  • Seasonal rules are automatically re-generated when the Auto Make Rules workflow runs.
  • See the Auto Make Rules Help for the generation workflow.
