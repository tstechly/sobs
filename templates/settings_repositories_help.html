Token Guidance
  • Minimum recommended scope for metadata/contents fetch is contents:read.
  • Validate token status after updates.

Onboarding Runbook
  1. Add app name and repository URL using the Add Repository wizard.
  2. Optionally set repo token and global fallback token.
  3. Run token validation to verify access and scopes.
  4. Add releases for CVE version timeline enrichment.
  5. Optionally set as default agent issue repository.
Troubleshooting
  • If repo data is not used in enrichment, verify repository URL format and default environment mapping.
  • If validation fails, inspect token scope and expiration state first.
  • If releases do not appear in scans, ensure app IDs and release tags are saved in this registry.
