Reports Help

Save, load, share, import, and export filter configurations as named reports.

Back to Reports
Save a Report

On any filter page (Logs, Traces, Errors, Metrics, RUM, AI, Work Items, Web Traffic), click Save as report in the filter bar to capture the current filters.

Apply a Report

Click Apply on any report card to navigate to the target page with the saved filters pre-applied.

Share & Export

Use the Share button to copy a shareable link, or Export to download reports as a JSON file for backup or transfer.

Report Fields
Field Description
Name A short human-readable label for the report, shown in the reports list and filter bar dropdown.
Description Optional notes about what the report captures or when to use it.
Page Type The target page: logs, traces, errors, metrics, rum, ai, work_items, or web_traffic.
Filters The saved filter key-value pairs. Applied as query parameters when the report is loaded.
Import & Export

Reports can be exported as a .json file and imported into another SOBS instance.

  • Export All — downloads all reports as a single JSON export file.
  • Export single — click the download icon on a report card to export just that report.
  • Import — click the Import button and upload a previously exported JSON file. Choose a conflict resolution strategy: rename, replace, or skip.
Loading Reports from Filter Pages

Filter bars on Logs, Traces, Errors, Metrics, RUM, AI, Work Items, and Web Traffic pages have a Load report dropdown. Select a saved report from the dropdown to instantly apply its filters without navigating to this page.
