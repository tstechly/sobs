Reference In-Cluster Setup

Follow the standard OTEL pattern used by ClickStack-style Kubernetes deployments: one deployment collector for cluster-wide signals and one daemonset collector for node-level signals.

  1. Deploy k8s/otel-k8s-deployment.yaml (single replica for cluster metrics/events).
  2. Deploy k8s/otel-k8s-daemonset.yaml (runs on each node for kubelet/log signals).
  3. Point exporter endpoint to SOBS OTLP endpoint (http://sobs:4318 in-cluster).

Once collectors are running, this dashboard reads from otel_metrics_* tables automatically.
