Access Control

The Table Explorer only exposes tables on the Query page allow-list. Internal SOBS configuration tables (prefixed with sobs_) are never shown.

  • All data access is read-only. No writes, updates, or deletes are possible.
  • Sample data is limited to a small number of rows per table (up to 10).
  • Additional tables can be added to the allow-list via the SOBS_QUERY_ALLOWED_TABLES environment variable.
Integration with the Query Page

The Table Explorer is linked from the Query page via the Explore Tables button in the page header. Use it to discover column names and data shapes before writing a natural-language query or custom SQL.
